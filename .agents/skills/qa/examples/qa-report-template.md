# QA 审查报告模板

> 本文件供 `qa` 技能在完成文档合规性全盘勘出后，格式化呈递检查结果。

## 审查档案
- **日期**：YYYY-MM-DD
- **审查目标**：`docs/development/issues/issue-123.md`
- **校验参照集**：所在目录的 `README.md`、全局 `.agents/rules.md`

---

## 缺陷拦截集 (Defects intercepted)

| 规则维度 | 发现的违规问题 | 修正要求与指引 | 严重度 |
|---------|--------|----------|--------|
| **命名规范** | 文件名的前缀使用了大写 `ISSUE-123.md` | 必须将其重命名为全小写前缀 `issue-123.md` | 🚨 Blocker |
| **排版结构** | Markdown 少填了 README 要求的 "规避方案" 块 | 严禁违反所在目录的排版指令，请补充该节内容 | 🚨 Blocker |
| **语言规范** | 代码演示块里的行内注释错用了中文 | 代码上下文和内连变量必须全部改为对应的原生英文 | ⚠️ Warning |

---

## 审查终局结论
- [ ] ✅ **Passed** — 命名的文件名称与所有内部结构排版均 100% 符合规范边界
- [ ] ❌ **Blockers** — 存在触碰红线的命名或格式结构破坏行为，已被打回并冻结后续流转
