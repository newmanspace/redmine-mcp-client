# Design Review Report 模板

> 本模板用于 `design-review` 技能输出架构评审报告时的参考格式。

## 评审摘要
- **日期**：YYYY-MM-DD
- **评审对象**：[Feature 文档名称 / 设计方案标题]
- **评审视角**：架构合规性、数仓分层合理性、MCP 规范一致性

---

## 评审维度

### 1. 数仓架构层级
- [ ] 数据流严格遵循 ODS → DWD → DWS → ADS 单向流转
- [ ] 无跨层数据抽取（如 ODS 直接供给 ADS）
- 评语：[...]

### 2. MCP 工具规范
- [ ] 所有新增接口命名符合 `<verb>_redmine_<noun>` 格式
- [ ] 入参类型定义清晰，返回值结构明确
- 评语：[...]

### 3. 性能与扩展性
- [ ] 增量快照逻辑合理（不会造成全量扫描）
- [ ] DWS 层聚合压力可控
- 评语：[...]

---

## 评审结论
- [ ] ✅ **Feasible** — 方案可行，无阻断项
- [ ] ⚠️ **Risks** — 方案存在风险点，需补充说明后可通过
- [ ] ❌ **Blockers** — 存在基础架构违规，需重新设计
