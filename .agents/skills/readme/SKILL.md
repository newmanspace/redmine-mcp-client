# README 生成器

本技能负责基于对当前项目真实代码库的深度扫描分析，自动生成一份高度专业、排版精美的 README.md 文档。

## 前置约束
- 取材必须基于对当前项目真实代码库的扫描分析。
- **绝对禁止**无中生有地编造项目中根本不存在的 Feature。
- 生成的内容必须与当前的代码库版本保持强一致。
- 排版格式要求高可读性（Scannable）和极简实用主义。

## 执行步骤

### 1. 分析项目结构
自动搜集上下文：
- 解析 `package.json`, `pyproject.toml` 或类似依赖清单。
- 顺带检查已有的旧文档资料。
- 摸清项目的核心目录结构并提取主要特性（Main Features）。

### 2. 梳理目标受众
明确阅读这份文档的人：
- 是给纯开发者看，还是给终端用户看？
- 他们接手这个项目需要优先了解什么？
- 是否有一些特定的公司/团队规范需要在文档中强调？

### 3. 构建文档骨架
输出的 README 将包含以下核心板块：
- **名称与简介**: 这是个什么项目，解决什么痛点。
- **标识组件 (Badges)**: 版本号、License、技术栈标签等。
- **快速开始 (Quick Start)**: 让人最快跑起来的核心几行命令。
- **环境安装 (Installation)**: 详尽的环境配置与依赖安装步骤。
- **使用指南 (Usage)**: 具体的代码运行或核心业务说明。
- **MCP API/命令行**: 向外暴露的工具簇与接口引用文档。
- **贡献指南 (Contributing)**: 内部开发流程与架构说明。

### 4. 撰写内容
在每个具体板块中：
- 废话少说，直击重点。
- 附带可直接运行的代码示例（Code Examples）。
- 如果有必要，留出版块粘贴 UI 截图。
- 给复杂的底层逻辑附加上跳转内部详细文档的链接。

### 5. 校验阶段
输出完毕后，提醒用户：
- 检查写上去的终端启动命令是否真的能在本地跑通。
- 检查文内的各种页面跳转链接是否全部有效。

## 核心法则
- 永远把"核心业务价值"放在第一段。
- 确保 Quick Start 真的足够 Quick。
- 所有代码示例必须做到"直接复制粘贴就能用"。
