# 代码审查 (Code Review)

本技能以项目 Committer 的严苛视角，对一段已完成的代码变更执行系统级的质量与安全审计。

## 审查检查点 (Checkpoints)
执行审查时，重点关注以下防呆列表：
1. **代码质量**：命名规范、单一职责、英文注释、是否有异常捕获。
2. **安全性**：是否存在 SQL 注入风险（必须参数化）、API 密钥是否硬编码、敏感日志是否脱敏。
3. **性能**：是否存在数据库 N+1 查询问题、IO 操作是否使用异步。
4. **覆盖率**：核心模块代码变更是否补充了对应测试，覆盖率是否下降。
5. **代码重复 (DRY)**：是否存在超过 5 行的相似代码块重复出现，是否应提取为工具函数。
6. **工具函数复用**：是否重复实现了已有的工具逻辑（如 `_calculate_next_send_at`）。

## 安全规范检查清单

### API 密钥管理
- **从环境变量读取**，禁止硬编码
- 禁止提交 `.env` 到 git（已在 `.gitignore`）

### SQL 注入防护
- **所有数据库查询使用参数化**
- 禁止字符串拼接 SQL

### 日志脱敏
- 敏感信息必须脱敏（密码、API 密钥、Token）
- 错误信息不泄露敏感数据

### SSL/TLS
- **生产环境禁止禁用 SSL 验证**

## 输出动作
自动根据以上列表审查对应代码，并输出一份结构化的 `Code Review Report`（参见 `code-review/examples/review-report-template.md`）：
包含 Critical/High/Medium 的风险分类，以及带有行号的具体修正建议。如果包含安全漏洞，将强制标记为阻断项。
