# 新特性开发 (Feature)

本工作流是整个项目产研流水线的**主控引擎**。当你需要开发一个全新的业务模块或独立接口时，触发此工作流。

## 目标与价值 (What it does)
它旨在将原本分散的架构设计、数据库审查、编码实现、测试验证等单点隔离的技能环节，拼装成一个**自动升降档的智能状态机**。
通过这一个统一入口，系统会在后台自动承接各个阶段的严格检查与红线防呆任务，最大限度降低开发者的心智与指令输入负担。系统仅会在关键的、不可逆的决策点（如架构最终敲定、代码合并前）将确认权交还给用户。

## 智能运作链路
触发该工作流后，系统将依次自主推进以下工作区块：

1. **构思与设计区块**
   - 调用 `design` 技能：主动收集项目技术栈环境并向你提问确认核心细节。
   - 自动生成符合项目规范的结构化 Feature 设计文档。

2. **双重内审区块**
   - 调用 `dba-review` 技能：在图纸阶段，对数据库 Schema 设计执行列顺序、主键选型等底层规范查杀。
   - 调用 `design-review` 技能：以 Committer 视角评估架构是否违背数仓分层原则与 MCP 命名规范。
   - 调用 `qa` 技能：对刚出具的设计图纸（Feature 文档）执行排版连贯性与命名小写化的结构安检。
   - **[人工放行点]**：将扫雷完毕的终版方案交给你，等待你的明确 Approve。

3. **受控编码区块**
   - 调用 `code` 技能：获得批准后，进入最严厉的执行模式，产出全英文业务代码，并强制套用 TDD 产出单元测试。
   - 调用 `code-review` 技能：完成后，进行代码白盒自审（查杀 SQL 注入、硬编码等）。

4. **验证与终局**
   - 调用 `test` 技能：自动拉起测试框架，跑完覆盖率清单。
   - **[人工放行点]**：汇总所有的战果与绿条，由你来拍板是否继续调用 `git-commit` 技能进行规范提交，并调用 `deploy` 技能部署上线。
